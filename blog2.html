Setting up Web Development environment on Ubuntu

Introduction

Lately I had to reinstall Ubuntu and I lost the configuration for the web development setup on my old machine. So, I thought I should put together a list of things that needs to be done to make the development workflow easier.

Get your system up to date

For updating Ubuntu, run the following commands in terminal 

sudo apt update && sudo apt upgrade

Install essentials

curl

Needed for API testing and for other network-related stuffs. 

sudo apt install curl

git

Needed for version control of your code 

sudo apt install git


You can configure it after that, so it doesn’t ask you for username all the time by typing these commands. Replace name and email with your github credentials. 

git config --global user.name "name"
git config --global user.email "email"

Google Chrome

Needed for testing web apps 

wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb
sudo apt install ./google-chrome-stable_current_amd64.deb

Node.js and npm

For better management of versions of npm and node, we will be installing nvm.

nvm is version manager for node.js, designed to be installed per-user, and invoked per-shell. It allows you to quickly install and use different versions of node via the command line.

nvm works on any POSIX compliant-shell (bash, zsh, fish, dash)

Run the following commands to setup nvm for the current user. 

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
export NVM_DIR="$([ -z "${XDG_CONFIG_HOME-}" ] && printf %s "${HOME}/.nvm" || printf %s "${XDG_CONFIG_HOME}/nvm")"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh" # This loads nvm

Verify installation by running the following command 

command -v nvm

Once installation is verified, run the following commands to install latest LTS versions of node and npm 

nvm install --lts

Check version of node and npm using the command 

nvm ls
node -v
npm -v

Visual Studio Code

The editor that is mostly used by developers is VS Code. It has got several extensions in the marketplace as well as a built-in terminal.

Run the following commands to install VS Code. 

wget -qO- https://packages.microsoft.com/keys/microsoft.asc | gpg --dearmor > packages.microsoft.gpg
sudo install -o root -g root -m 644 packages.microsoft.gpg /etc/apt/trusted.gpg.d/
sudo sh -c 'echo "deb [arch=amd64,arm64,armhf signed-by=/etc/apt/trusted.gpg.d/packages.microsoft.gpg] https://packages.microsoft.com/repos/code stable main" > /etc/apt/sources.list.d/vscode.list'
rm -f packages.microsoft.gpg
sudo apt install apt-transport-https
sudo apt update
sudo apt install code

Few extensions which I install to make it better are:-
  • Prettier — great if you want to keep your code consistent and neat. You can customise it to work after saving by going to File>Preferences>Settings and looking for ‘Editor:Format On Save’. While you’re there you can also change auto-save settings to be triggered by onFocusChange.
  • Bracket pair colorizer — just very useful. You can never trust yourself with bracket matching.
  • REST Client — REST Client allows you to send HTTP request and view the response in Visual Studio Code directly.
  • Live Server — Launch a development local Server with live reload feature for static & dynamic pages
  • GitLens — Git supercharged — GitLens simply helps you better understand code. Quickly glimpse into whom, why, and when a line or code block was changed. Jump back through history to gain further insights as to how and why the code evolved. Effortlessly explore the history and evolution of a codebase.
  • HTML CSS Support — CSS Intellisense for HTML
